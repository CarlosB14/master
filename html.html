    Tablas
    • Table --> Engloba todo el contenido de la tabla
    • tr --> para flias
    • th --> para celdas de cabecera, negrita y semántica, van dentro de tr
    • td --> celdas normal que no son de cabecera, va dentro de tr
    • caption --> Justo debajo de table. Para el título de la tabla, encima o bajo de la tabla
    • border --> sería table border="1"> Le damos borde a la tabla
    • Atributo colspan --> Fusionar celdas adyacentes td colspan="2" y no pondríamos el siguiente td
    • Atributo rowspan --> Fusiona flias igual que las celdas
    • thead --> Define donde empieza y acaba la cabecera
    • tbody --> Engloba el cuerpo de la tabla
    • tfoot --> Normalmente se define después de thead y antes de tbody

    Formularios
    • form --> Envuelve todo el formulario
      • Atributo action --> Dirección URL del backend donde se envía la información
      • Atritubo method--> Se usará con GET: Sólo form consulta, no modf ni crear registros, sólo 500 bytes no archivos. POST: De forma ocultapara, envío muchos datos, sí archivos, tmb creación eliminación o borrado
      • Atritubo name--> Para identificar un formulario
      • Atritubo target--> Para abrir en una pestaña nueva con _blank
      • Atritubo enctype--> Esencial para cuando se envíe un archivo. Para determinar en que codificación van a viajar los ficheros
      • Atritubo accept-charset--> Fuerza a utliizar una codificación en los parámetros del formulario
      • Atritubo autocomplete--> On o Off y es para que el navegador pueda usar datos que se han enviado en otros formularios
      • Atritubo novalidate--> No lo aconseja es para desacativar la validación del formulario
        • Input --> Elemento donde podremos escribir la información que queremos enviar al servidor, según la naturaleza de los datos usaremos uno o otro
        • Input--> type = --> button-Crea botón, checkbox, color, date, datetime-local, emali-obliga a emali, flie-para adjuntar ficheros obligado atributo enctype, hidden-enviar datos al servidor sin ver usuario, image, month, number-sólo numeros, password-sale cifrado, radio, range, reset-borra formulario, search, submit, tel, text-sólo texto, time, url-obliga a url, week
        • name--> Es la clave valor, muy importante en todos los inputs le añadimos la clave con la que queremos trabajar nosotros cada interacción del usuario
        • type="checkbox"--> Input de cosas que el usuario podrá seleccionar o no, puede elegir las que quiera
        • type="color"--> Para que el usuario seleccione un color
        • type="date"--> Nos añade input para seleccionar fecha
        • type="datatime-local"--> Nos une fecha y hora en un input
        • type="image"--> Para usar una imagen de envío, cómo una imagen con vínculo pero para enviar la petición del formulario
        • type="month"--> Input de mes
        • type="radio" value="hombre"--> Input de por ejemplo elige sexo, sólo puede elegir uno. Importante mismo name para que si eliges uno o otro deseleccione el anterior
        • type="range" min="0" max="10"-->Input de un rango del 0 al 10
        • reset-->
        • type="search"--> En la caslila de búsqueda añade una x para eliminar el contenido escrito
        • type="tel"--> Input de teléfono
        • type="time"--> Input para seleccionar una hora
        • type="week"--> Input de semana
        • id=""--> No pueden haber dos iguales en el documento, le estamos dando una identidad
        • placeholder--> Sale en el recuadro del input un texto nuestro indicando cualquier cosa, en cuanto escriba desaparece
        • required--> Se puede usar en todos los inputs, y es para obligar al usuario a rellenarlo para enviar
      • label for="id"--> Hace que el texto que envuelve label tenga accesibliidad con, por ejemplo, el radio, y al pulsar el texto tmb seleccione la caslila
      • textarea cols="" rows=""--> Crea zona para escribir texto de x columnas y x flias
        • select name="" id=""--> Para hacer desplegable de varias opciones por ejemplo tallas
        • option value="" required--> Empezar el option con value vacío. Rodeamos el texto con option de lo que queremos que aparezca en el desplegable
        • select multiple--> Te salen todas las opciones para seleccionar una de ellas en el desplegable
      • fieldset--> Engloba en una caja diferentes campos de un formulario para poder organizarlos
      • legend--> En fieldset le damos definición a ese grupo de inputs del formulario
    Imágenes en web
    • Imágenes contenido--> Las implantaremos con html
    • Imágenes presentación o decoración--> Las implantaremos con CSS
      • img--> Etiqueta vacía para imáegenes, se complementa todo con atributos
      • src--> Es la url donde está la imagen que vamos a mostrar, obligado
      • alt--> Texto alternativo explicando la imagen para navegadores que no la puedan mostrar. Obligado tmb
      • with--> Ancho de la imagen
      • height--> Alto de la imagen
      • loading--> Tiene como opción por defecto eagereager, descarga todas las imágenes. LazyLazy el navegador pospone la descarga hasta que es visible.
      • figure--> Agrupación de imágenes, pones la imagen dentro
      • figcaption--> Es la leyenda de figure, le da semántica
      • srcset--> Le damos al navegador las imágenes en diferente ancho o densidad de píxeles para que carge una o otra en relación donde se está visualizando
      • sizes-->
      • picture--> Agrupa serie de imágenes
      • source--> Mostrará la imagen que cumpla una serie de criterios
      • source media="(with < 100px)"--> Podemos hacer condicionales para que muestre una imagen o otra
