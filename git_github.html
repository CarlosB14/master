Git y Github

    Git bash

  • git init--> En la carpeta que estamos iniciamos git
  • git status--> Vemos que está pasando
  • git add .--> Añadimos todos los archivos modificados o creados a git
  • git commit -m "message"--> Guardamos las modificaciones o creaciones con un mensaje para saber cambios hechos
  • git log--> Nos muestra los pantallazos hechos
  • git checkout hash--> Nos movemos al hash indicado
  • git log --graph --decorate --all--> Muestra log en una línea simple
  • git reset--> No te deja hacerlo si tienes archivos sin guardar
  • git config --global alias.tree "comando"--> Macro con nombre tree del comando registrado
  • git ignore--> Ignorar archivo en status o carpeta
  • git reset --hard --hash--> Elimita lo de después de este hash
  • git reflog--> Muestra todos los hash y movimientos
  • git reset --hard último-hash--> Nos sitúa en el último hash aún eliminándolo
  • git tag--> Para marcar un pantallazo con más énfasis
  • git checkout tags/nombre-tag--> Nos lleva a ese tag
  • git tag--> Nos dice los tags creados
  • git branch nombre--> Crear una rama desde donde estamos
  • git switch rama--> Nos movemos a otra rama
  • git merge rama--> Nos une la información desde la rama donde estamos a la que hacemos el merge normalmente master/main
  • git stash--> Congela lo que hacemos y nos permite cambiar de rama sin hacer commit
  • git stash list--> Nos muestra los slash abiertos
  • git stash pop--> Vuelves al slash como estaba
  • git stash drop--> Elimina el slash
  • git diff rama--> Te dice si hay cosas nuevas en esa rama o cambios
  • git branch -d rama--> Eliminamos rama, si hemos hecho merge con main, no pasa nada porque ya tenemos esos cambios mergeados en el main
  • git branch -m main--> Nos cambia de nombre la rama por uno que queramos, master por main

      • Github con git bash

  • git remote add origin git@github.com:CarlosB14/repositorio.git--> Nos vincula git con github
  • git push -u origin master/main--> Para subir archivos por primera vez, luego sólo git push
  • git fetch--> Descarga historial de cambios y lo podemos ver en git tree IMPORTANTE para hacer git push
  • git pull origin master/main--> Hará merge por defecto IMPORTANTE si hay cambios y querer hacer git push
  • git clone "url ssh de github del repositorio a clonar"--> Clonamos repositorio mediante ssh
